========================================================================
             СПРАВКА: РАБОТА МОДУЛЕЙ "ПОД КАПОТОМ" (ABOUT)
========================================================================

В данном документе приведено детальное техническое описание внутренней
логики работы каждого модуля программного комплекса Linux Security Checker.

------------------------------------------------------------------------
1. Главный хаб управления (hub.sh)
------------------------------------------------------------------------
* Инициализация TUI: Экспортирует переменную среды `NEWT_COLORS` для 
  задания единой темы интерфейса whiptail (цвета НИИ «Рубин»).
* Проверка привилегий: Сравнивает UID текущего пользователя с нулем 
  (проверка на root/sudo). В случае несоответствия прерывает запуск.
* Мониторинг зависимостей: Использует `dpkg-query` для проверки статуса 
  пакетов openscap-scanner, openscap-common и unzip в базе APT.
* Диспетчеризация: После выбора действия проверяет наличие исполняемого 
  файла скрипта в `scripts/`, автоматически устанавливает флаг 
  исполнения (`chmod +x`) и запускает процесс.

------------------------------------------------------------------------
2. Аудит безопасности и съемка скриншотов (scripts/security_checker.sh)
------------------------------------------------------------------------
* Системный аудит: Опрашивает конфигурационные файлы ОС:
  - sysctl (параметры ядра, сетевой стек, защита адресного пространства).
  - PAM (/etc/pam.d/) и политики паролей (/etc/login.defs).
  - Настройки SSH-сервера, UFW/iptables и активные сетевые порты.
  - Атрибуты безопасности Astra Linux (Parsec MAC/MIC, состояние ОРД).
* Съемка скриншотов:
  - Определяет адрес X-сервера (по умолчанию DISPLAY=:0) и извлекает 
    контекст авторизации (.Xauthority) для возможности работы графических 
    приложений от имени root.
  - Последовательно запускает приложения панели управления Fly/KDE, 
    указанные в `service/screenshot_apps.list` (или полученные динамически 
    через `astra-systemsettings --list`).
  - Использует утилиту `scrot` для захвата окна графического приложения, 
    сохраняя файлы в `report/screenshots/`.
  - Завершает процесс приложения после захвата кадра.
* Генерация отчета: Вызывает `service/report_generator.sh` для сборки 
  интерактивного HTML-отчета на основе собранных текстовых данных (сохраняется в `report/security_report.html`), а также сохраняет список установленного ПО в `report/installed_software.txt`.
* Контроль соответствия классу защищенности:
  - Оценивает настройки безопасности ОС на соответствие требованиям РД АС и РД СВТ для выбранного класса (3Б, 2Б, 1Д, 1Г, 3А, 2А, 1В, 1Б, 1А).
  - Считывает эталонные параметры (длина/история/срок действия паролей, ограничения попыток входа, таймауты, SecDel, SwapWiper, console-lock и т.д.) из редактируемого конфигурационного файла `service/compliance_standards.conf`.
  - Формирует подробный текстовый отчет с рекомендациями по устранению несоответствий и путями в Панели управления Fly. Отчет сохраняется в файл `report/compliance_report_[КЛАСС]_[ДАТА_ВРЕМЯ].txt`.

------------------------------------------------------------------------
3. OVAL Сканирование уязвимостей (scripts/scanoval_checker.sh)
------------------------------------------------------------------------
* Подготовка баз: Ищет ZIP-архивы OVAL-определений ФСТЭК на смонтированных 
  накопителях или скачивает их напрямую с БДУ ФСТЭК (при наличии сети).
* Запуск OpenSCAP: Вызывает сканер OpenSCAP с параметрами:
  `oscap oval eval --results results.xml --report report.html definition.xml`
* Кастомизация отчета: Запускает парсер `service/generate_custom_report.py`, 
  который пересобирает стандартный XML-результат OpenSCAP в более 
  информативный и адаптивный графический HTML-отчет в каталоге `report/scanoval/`.

------------------------------------------------------------------------
4. Установка антивирусного ПО (scripts/antivirus_installer.sh)
------------------------------------------------------------------------
* Автоматический поиск: Сканирует смонтированные файловые системы (/media, 
  /mnt, /run/media) и корень проекта на наличие установочных файлов 
  антивирусов по маскам имен (kesl*.deb/kesl-gui*.deb для DEB-систем, kesl*.rpm/kesl-gui*.rpm для RPM-систем, а также drweb-workstations*.run/drweb-file-servers*.run).
* Установка: Предоставляет выбор между Kaspersky Endpoint Security и 
  Dr.Web Security Space. Если инсталлятор найден, запускает установку с 
  выводом прогресса на whiptail-gauge и логированием в /tmp/antivirus_install.log. 
  Если файл не найден, позволяет пользователю указать путь вручную.
* Обновление баз (офлайн): Позволяет настроить локальный источник баз (указав папку с базами на флешке/диске) и запустить задачу обновления без интернета.
* Активация лицензии: Ищет файлы ключей (`*.key`) на съемных дисках или позволяет ввести путь к ключу вручную. KESL активируется через `kesl-control --add-active-key`, а для Dr.Web файл копируется в `/etc/opt/drweb.com/drweb32.key` и конфигурация перезапускается командой `drweb-ctl reload`.
* Унифицированная обратная связь: При успешном выполнении любых операций (установка, обновление, активация) выводится окно успеха, а при ошибках — окно просмотра лога.

------------------------------------------------------------------------
5. Менеджер локального репозитория (scripts/repo_manager.sh)
------------------------------------------------------------------------
* Монтирование: Монтирует указанный пользователем ISO-образ или CD-ROM 
  в каталог `/media/astra_iso` в режиме "только чтение" (`mount -o loop,ro`).
* Изоляция сети: Временно комментирует все интернет-репозитории в файлах 
  `/etc/apt/sources.list` и `/etc/apt/sources.list.d/*.list`, помечая 
  их префиксом `# OFFLINE_DISABLED:`.
* Подключение локального источника: Добавляет запись локального пути 
  `deb [trusted=yes] file:/media/astra_iso/ ...` в sources.list.
* Обновление кэша: Выполняет `apt-get update` для перестроения базы APT.
* Демонтирование и откат: Размонтирует образ, удаляет запись локального 
  источника и снимает комментарии с сетевых репозиториев.

------------------------------------------------------------------------
6. Настройка мандатного доступа и управление системой (scripts/hub_sys.sh)
------------------------------------------------------------------------
* Настройка мандатных уровней (MAC):
  - Позволяет пользователю выбрать максимальный уровень мандатного доступа от 0 (Несекретно) до 3 (Сов.секретно).
  - Поддерживает два режима именования уровней: по ГОСТ (Несекретно, ДСП, Секретно, Сов.секретно) или с ручным вводом собственных названий.
  - Перезаписывает конфигурационный файл `/etc/parsec/mac_levels`.
  - Выполняет перезапуск службы `parsec.service` и инициализирует настройки безопасности через `/usr/lib/parsec/systemd/parsec_systemd_init`.
* Диспетчеризация управления пользователями:
  - Предоставляет подменю с выбором действий по администрированию учетных записей, запуская соответствующие вспомогательные скрипты (`Создание пользователей.sh`, `Создание папок для пользователей.sh`, `Удаление пользователя.sh`).

------------------------------------------------------------------------
7. Создание пользователей с МКД (scripts/Создание пользователей.sh)
------------------------------------------------------------------------
* Чтение мандатной структуры: Парсит системные файлы безопасности 
  `/etc/parsec/mac_levels` для получения доступных мандатных уровней.
* Создание аккаунта: Вызывает бинарный файл `useradd` с флагами GECOS 
  (ФИО, должность) и перечнем дополнительных системных групп.
* Конфигурация Parsec MACDB: Генерирует строку мандатных параметров в формате:
  `username:min_lvl:min_cat:max_lvl:max_cat:integrity_level:flags`
  Записывает эту строку в персональный конфигурационный файл мандатного 
  доступа в `/etc/parsec/macdb/<UID>` с правами 600 (owner: root).
* Контроль сложности пароля: Извлекает и парсит конфигурацию PAM 
  `/etc/pam.d/common-password` для отображения действующих лимитов (minlen, 
  credits). Задает пароль через команду `chpasswd`.

------------------------------------------------------------------------
8. Создание папок для пользователей (scripts/Создание папок для пользователей.sh)
------------------------------------------------------------------------
* Фильтрация пользователей: Выбирает из `/etc/passwd` реальных пользователей 
  (UID >= 1000) с кириллической фамилией в поле GECOS.
* Иерархия каталогов:
  - Создает родительский каталог в `/home/Документы/<Фамилия_пользователя>`.
  - Создает подкаталоги для каждого мандатного уровня доступа, не 
    превышающего максимальный уровень пользователя (напр., "Открыто", "Секретно").
* Мандатная разметка: Вызывает Astra Linux утилиту `pdpl-file` для установки 
  мандатных уровней доступа (уровень секретности, целостность) на каждый 
  каталог индивидуально, обеспечивая изоляцию данных по правилам MAC/MIC.

------------------------------------------------------------------------
9. Удаление пользователя (scripts/Удаление пользователя.sh)
------------------------------------------------------------------------
* Терминация процессов: Принудительно завершает все активные сессии и 
  процессы пользователя командой `pkill -u <username>` для снятия блокировки.
* Удаление аккаунта: Выполняет `userdel -r -f <username>`, очищая запись 
  в passwd/shadow и стирая домашний каталог в `/home/<username>`.
* Очистка Parsec: Удаляет конфигурационный файл пользователя `/etc/parsec/macdb/<UID>`.
* Очистка документов: Удаляет каталог документов в `/home/Документы/<Фамилия_пользователя>`.
